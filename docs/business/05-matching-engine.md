# 撮合引擎（Matching Engine）核心算法

撮合引擎是 CEX 的核心模块，负责将买卖委托按规则匹配并成交。撮合过程中的**余额处理**与账户系统一致：

```
创建订单 → 冻结对应余额
订单结束 → 解冻、扣款、退还未使用余额
```

> 例：一笔买单最终实际消耗 300 USDT，则冻结 300 USDT，成交后按实际成交额扣减，剩余原路退回。

---

## 订单簿与撮合的关系

撮合引擎依赖 [订单簿](./04-order-book.md) 中的买卖队列：

| 队列 | 排序规则 | 最优价 |
| :--- | :--- | :--- |
| **卖盘（asks）** | 价格由低到高 | 最低卖价（卖一） |
| **买盘（bids）** | 价格由高到低 | 最高买价（买一） |

**成交条件：** 买单委托价 ≥ 卖单委托价（即买价覆盖卖价）。

订单簿记录的是尚未成交的**限价挂单**；市价单不进入订单簿排队，直接与对手盘最优价撮合。

---

## 撮合优先级

| 优先级 | 规则 |
| :--- | :--- |
| **价格优先** | 卖盘低价优先、买盘高价优先 |
| **时间优先** | 同价位按提交时间先后成交（FIFO） |

## 限价单撮合

### 案例 1：限价买单无法成交 → 进入买盘排队

**当前卖盘：**

| 价格 | 订单 | 数量 |
| :--- | :--- | :--- |
| 103.0 | A | 100 |
| 103.0 | B | 200 |
| 103.5 | C | 150 |
| 104.0 | — | 300 |

**新订单：** 限价买单，委托价 **102.8**，数量 **180**

卖盘最优价 103.0 **>** 买单委托价 102.8，不满足成交条件 → 买单进入**买盘队列**排队（买盘高价优先，102.8 排在对应价位首位）。

### 案例 2：限价买单逐级吃单

**新订单：** 限价买单，委托价 **103.8**，数量 **350**

103.8 ≥ 103.0，满足条件，从最优卖价逐级匹配：

| 步骤 | 匹配对象 | 成交量 | 剩余可买 |
| :--- | :--- | :--- | :--- |
| 1 | 103.0 订单 A | 100 | 250 |
| 2 | 103.0 订单 B | 200 | 50 |
| 3 | 103.5 订单 C | 50（C 剩余 100） | 0 |

350 枚全部成交。更新后卖盘：103.5 价位剩余 100 枚。

## 市价单撮合

市价单**不指定价格**，以市场当前最优价成交：

| 类型 | 匹配方向 | 用户填写 |
| :--- | :--- | :--- |
| **市价买单** | 卖盘最优价（最低卖价） | 投入总金额（USDT） |
| **市价卖单** | 买盘最优价（最高买价） | 卖出代币数量 |

### 案例 3：市价买单（按金额）

**卖盘同上。** **新订单：** 市价买单，总金额 **30,000 USDT**

| 步骤 | 匹配对象 | 计算 | 剩余金额 |
| :--- | :--- | :--- | :--- |
| 1 | 103.0 订单 A（100 枚） | 100 × 103 = 10,300 | 19,700 |
| 2 | 103.0 订单 B（200 枚） | 全额需 20,600 > 19,700，可买 ⌊19700÷103⌋ = **191** 枚 | 19,700 − 191×103 = **27** |

剩余 27 USDT 不足以匹配下一档（103.5），对应未完成部分进入队列等待后续撮合。

### 案例 4：市价卖单（按数量）

**当前买盘：**

| 价格 | 订单 | 数量 |
| :--- | :--- | :--- |
| 102.5 | A | 150 |
| 102.5 | B | 100 |
| 102.0 | C | 200 |

**新订单：** 市价卖单，卖出 **400** 枚

| 步骤 | 匹配对象 | 成交量 | 剩余待卖 |
| :--- | :--- | :--- | :--- |
| 1 | 102.5 订单 A | 150 | 250 |
| 2 | 102.5 订单 B | 100 | 150 |
| 3 | 102.0 订单 C | 150 | 0 |

成交汇总：102.5 价位 250 枚 + 102.0 价位 150 枚，共 400 枚。总成交额 = 150×102.5 + 100×102.5 + 150×102.0 = **40,925 USDT**。

## 订单流转总结

```
订单提交
  │
  ├─ 限价单 → 优先与对手盘限价单匹配
  │            ├─ 完全成交 → 更新订单簿、结算账户
  │            └─ 部分成交 → 剩余继续与市价单匹配（加速消化）
  │
  └─ 市价单 → 直接按最优价与对手盘撮合
               └─ 未完全成交部分 → 进入队列等待

多轮匹配后仍有剩余 → 写入对应盘口队列，等待新订单
```

**第二轮跨类型匹配：** 未完全成交的限价单，会继续与场内市价单撮合。市价单用户对价格无严格要求，有助于快速消化遗留挂单，提升整体撮合效率。

---

## 成交后的清算

每笔撮合成功后，引擎触发：

1. **更新订单簿** — 扣减或移除已成交挂单
2. **更新账户** — 解冻、扣款、增加买到的资产；扣除手续费
3. **生成成交记录** — 写入 trade 表，推送至前端

## 本章小结

| 订单类型 | 匹配规则 | 未成交处理 |
| :--- | :--- | :--- |
| **限价买单** | 委托价 ≥ 卖盘最优价时，从低价卖单逐级吃单 | 进入买盘队列 |
| **限价卖单** | 委托价 ≤ 买盘最优价时，从高价买单逐级吃单 | 进入卖盘队列 |
| **市价买单** | 从最低卖价开始，按金额逐档吃单 | 剩余金额排队 |
| **市价卖单** | 从最高买价开始，按数量逐档吃单 | 剩余数量排队 |
